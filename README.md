# yutong.design

Personal portfolio rebuild for Yutong Chen.

## Getting Started

Run the development server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Content

Homepage and project copy live in:

- `src/content/site.ts`
- `src/content/projects.ts`

## Homepage Voice

Keep the portfolio warm, direct, and handcrafted:

- Use lowercase casual language where it feels natural: "hey," "i'm yutong".
- Use kaomoji sparingly as warm punctuation: `◝(ᵔᵕᵔ)◜`, `•ᴗ•`, `:-)`.
- Preserve distinctive symbols for labels and wayfinding: `⚑ Currently`, `➭ Prev.`, `→ Email`, `☟`.
- Keep section casing intentional: Title Case section headers, lowercase nav items.
- Prefer personal sign-offs like "Thank u for visiting!" and "Let's connect :-)" over generic footer language.

## Homepage Structure

Structural choices from the Framer reference worth preserving:

- Keep the hero information-dense: greeting, name, current role, previous roles, and resume CTA above the fold.
- Place "Featured Projects" as the second beat, with a small downward cue.
- Let the short bio bridge the project heading and the project grid.
- Show four featured projects in a 2x2 typography-led grid.
- Keep project cards simple: tags, title, client, and date.
- Preserve three footer zones: personal sign-off, copyright, and contact links.
